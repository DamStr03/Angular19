Théorie - Intro

Commande de création du projet :

Commande : ng new demo-angular --directory . --skip-tests

Où :

  1. `ng new` : Initialise un nouveau projet Angular

  2. `demo-angular` : Nom du projet

  3. `--directory .` : Installe le projet dans le répertoire actuel sans recréer un dossier supplémentaire

  4. `--skip-tests` : Empêche la génération de fichiers de tests

Démarrer son serveur :

ng serve : Démarre le serveur (le paramètre --o peut être ajouté pour ouvrir le projet sur une page web).

Le processus Ahead-of-time (AOT) :

Technique de compilation utilisée par Angular pour améliorer les performances des applications.

L'AOT compile l'application avant son exécution, convertissant le code Angular en JavaScript optimisé avant que le navigateur ne le charge. Cela rend le chargement initial plus rapide et réduit la taille des fichiers transmis au navigateur.

C'est aussi ce processus qui permet à l'application de se recharger automatiquement à chaque modification sans devoir relancer le serveur.

Analyse de l'arborescence initiale du projet :

  1. `node_modules` : Contient tous les modules nécessaires au framework

  2. `src` : Contient le code de notre projet

  3. `angular.json` : Configuration de l'espace de travail du projet

    • C'est ici que tu trouveras les configurations définies lors de la création du projet, comme --skip-tests.

  4. `package.json` : Listes & versions des packages du projet

  5. `tsconfig.json` : Configuration de la compilation TypeScript

La programmation en Angular :

1. Les différents paradigmes de programmation :

Angular implémente trois paradigmes principaux de programmation :

  1. Le paradigme orienté objet

  2. Le paradigme fonctionnel

  3. Le paradigme réactif : Angular adopte fortement le paradigme réactif pour gérer les événements asynchrones, comme les requêtes HTTP, les interactions utilisateur ou les changements d'état.

    • Grâce à ce paradigme, les données deviennent réactives. Lorsqu'une donnée change, toutes les parties du programme dépendant de cette donnée sont mises à jour automatiquement.

    • Exemple : Lors d'un changement de données, l'interface utilisateur se met à jour automatiquement, déclenchant des actions en fonction des modifications.

2. L'architecture en Angular :

Angular est basé sur une architecture de composants. Chaque composant est une partie de l'interface utilisateur qui peut être réutilisée et encapsulée dans un seul élément.
